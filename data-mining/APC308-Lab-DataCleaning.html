1. Data Cleaning Basics¶

  • You may remember the GIGO (Garbage In, Garbage Out) concept in programing. The quality of the input determines the quality of the output.
  • Often times the real-world data is huge, messy and unclean. Doing data analytic or mining operations on dirty data could lead to an unreliable and incorrect result.
  • A number of factors could contribute to an unclean data including data entry errors, poorly designed forms for data collection, duplicate data, inconsistent data formats, outdated data (data decay), etc.
  • The data cleaning process attempts to enhance the quality of the data by filling missing values, getting rid of inconsistencies, smoothing out noise (a random error in a measured variable).
  • Data cleaning can be seen as one form of data preprocessing along with other operations such as data integration, data reduction, data transformation, etc. You may recall our discussion on the KDD process shown in the figure below.
  • In this lab session, we will focus on some common data cleaning operations that would help us to preprate our data for reliable mining.

Visualization of Missing values¶

  • There are useful libraries that help in the visual inspection of missing values. One such library is missingno.
  • The library provides the following plot types:
  1. Matrix plot: helps us to better visualize missing values.
  1. Bar plot: count of values present in columns excluding missing values.
  1. Heatmap: shows nullity correlation between columns of the dataset. The stronger the blue, the more the correlation. The correlation ranges from -1 to 1. -1 is negative correlation (if one value is present, the other is missing), 0 (no correlation), and 1 (positive correlation).
  1. Dendrogram: shows the hierarchical relationship between attributes by grouping together (clustering) columns that have more nullity relation. Attributes that have stronger connection in missing values between them will be kept in the same cluster.
  • The library can be installed using pip or conda. Go ahead and do so if not installed already.

2.2 Dealing with Missing Values¶

There are a number of approaches to deal with missing values. Each has its pros and cons. We shall see selected methods in this lab session:

  1. Dropping the attribute/instance with missing values. This should be used carefully (e.g. when a column has many missing values).
  1. Filling missing values with a global constant such as 0, 'unknown', 'missing', etc.
  1. Imputing (substituting) missing values with compensation values such as mean and median.
  1. Filling missing values with the most probably value using techneiques such as regression. More on regression later in this module.
